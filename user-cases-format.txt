Use Case: Adicionar Campeonato
Cenário: Campeonatos
Pré-Condição: Utilizador deve estar autenticado como administrador
Pós-Condição: Campeonato adicionado ao sistema
Fluxo-Normal:
  1- Administrador fornece o nome do Campeonato
  2- Sistema valida o nome do Capeeonato
  3- Sistema apresenta lista de Circuitos
  4- Administrador seleciona os circuitos que deseja
  5- Sistema adiciona campeonato à lista de campeonatos disponíveis
Fluxo-Exceção(1)(Nome do Circuito já existe(Passo 2)):
  2- Sistema verifica que o nome de campeonato já existe
  3- Sistema informa utilizador da já existência desse nome


Use Case: Adicionar um novo Circuito
Descrição: Administrador decide adicionar um novo circuito no sistema
Cenário: Circuitos
Pré-Condição: Utilizador deve estar autenticado como Administrador
Pós-Condição: Circuito adicionado à lista de circuitos
Fluxo Normal:
  1- Administrador fornece o nome do circuito a adicionar
  2- Sistema verifica que nenhum circuito existe com o nome fornecido
  3- Administrador fornece distancia, curvas e chicanes do circuito
  4- Sistema calcula o número de retas do circuito
  5- Sistema fornece lista de retas e curvas do circuito, para que seja introduzida a dificuldade
  6- Utilizador fornece para cada reta e curva a dificuldade desejada
  7- Utilizador diz por quantas voltas o circuito será constituído
  8- Sistema adiciona circuito à lista de circuitos
Fluxo Exceção[Nome do circuito já existe](Passo 2):
  2- Sistema verifica que nome do circuito informando o ator 

Use Case: Adicionar novo piloto
Descrição: Administrador decide adicionar um novo piloto ao sistema
Cenário: Pilotos
Pré-Condição: Utilizador deve estar autenticado com Administrador
Pós-Condição: Piloto adicionado ao sistema
Fluxo-Normal:
  1- Administrador fornece o nome do piloto a adicionar
  2- Sistema verifica que nenhum piloto existe com o nome fornecido
  3- Sistema pede ao utilizador para inserir os níveis de perícia(CTS e SVA)

  6- Sistema adiciona piloto ao sistema
Fluxo Exceção 1(Nome do piloto já existe(Passo 2)):
  2- Sistema verifica que nome do piloto que ator forneceu já existe

Use Case: Adicionar Carro

Use Case: Alterar Password
Descrição: Ator decide alterar o seu password
Cenario:
Pré-Condição: Ator deve estar autenticado no sistema
Pós-condição: Ator terá a sua password alterada
Fluxo-Normal:
  1- Ator fornece nova password
  2- Sistema valida nova password
  3- Sistema altera a password do ator

Fluxo-Exceção[password fornecida já era a password do utilizador](Passo 2):
  2- Sistema verifica que password do utilizador era igual à fornecida informando o Ator
—> Se não for acrescentar muita complexidade, podiam colocar aqui uma validação da password (p.ex: mínimo de caracteres: e se o utilizador colocar uma password vazia?)
Fluxo-Exceção(password inválida(vazia ou caracteres não satisfeitos)(Passo 2)):
  2- Sistema informa que password nova não é valida

Use Case: Jogar Campeonato
Descrição: Jogador decide jogar campeonato
Cenário: Jogar
Pré-Condição: Sistema não se encontra a executar nenhum campeonato, tendo neste momento um lobby criado
Pós-Condição: Campeonato terá decorrido atribuindo pontuação a cada jogador autenticado
Fluxo-Normal:
  1- <<include>> Inscrição em Campeonato
  2- Jogador espera que todos os jogadores se inscrevam
  3- Sistema apresenta condições da corrida, circuito e situação meteorológica
  4- <<include>> Configuração do Carro para Corrida
  5- Jogador espera que todos os jogadores do lobby configurem o carro
  6- Sistema inicia simulação da corrida
  7- No Final da simulação, sistema apresenta resultado da corrida
  8- Sistema soma a pontuação de cada jogador na corrida à sua pontuação no Campeonato
  9- Para cada circuito restante, repetir Passos(7-13)
  10- No Final do Campeonato sistema apresenta classificação final para as diferentes categorias
  11- Sistema adiciona pontuação do jogador no campeonato ao seu ranking global 
  12- Sistema adiciona participação ao histórico do jogador
Fluxo-Alternativo[Jogador não se encontra autenticado mas tem conta e deseja receber pontuação](Passo 12):
  12.1- Sistema verifica que utilizador não se encontra autenticado e pergunta se utilizador tem conta
  12.2- Jogador diz que sim
  12.3- <<include>> Autenticação
  12.4- Regressa ao Passo 12
Fluxo-Alternativo[Jogador não se encontra autenticado e não tem conta](Passo 12):
  12.1- Sistema verifica que utilizador não se encontra autenticado e pergunta se utilizador tem conta
  12.2- Jogador diz que não

Use Case: Configuração de Carro para Corrida
Descrição: Ator decide efetuar a configuração do carro que vai usar
Cenário: Jogar.Configurar Corridas
Pré-Condição: Campeonato iniciado, com corrida escolhida
Pós-Condição: Ator pronto a iniciar corrida, com o carro devidamente configurado a seu dispor
Fluxo-Normal:
  1- Sistema avisa ator que deve escolher um motor e os pneus do veículo
  2- Sistema fornece os possíveis modos de motor 
  3- Ator escolhe um modo de motor
  4- Sistema guarda modo de motor
  2- Sistema fornece tipos de pneus possíveis
  6- Ator escolhe tipo de pneus
  7- Sistema guarda sua escolha
Fluxo-Alternativo(Ator possui carro dos tipos C1 ou C2, pretende alterar afinação e ainda não realizou o máximo de afinações permitidas- Passo 0):
  0.1- Ator decide valor para a afinação do carro
  0.2- Sistema efetua mudança
  0.3- Regressa a 1

Pós-Condição: Sistema terá criado utilizador
Fluxo-Normal:

  2- Sistema valida username
  3- Utilizador fornece password

Fluxo-Exceção[password invalida(p.e vazia)](Passo 4):
